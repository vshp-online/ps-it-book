# Теория вероятностей и математическая статистика для начинающих IT-предпринимателей (учебное пособие)

Исходники учебного пособия по теории вероятностей и математической статистике в формате **Quarto Book** и сопровождающие материалы: примеры кода, данные, рабочие книги LibreOffice Calc и иллюстрации.

> Книга основана на многолетних наработках авторов; работа над ней началась более года назад и сейчас переносится из исходного формата в электронный Quarto. Быстрые изменения в главах связаны с переносом уже подготовленного материала.

Авторы: Н.Н. Зубов, П.С. Ткачев, С.З. Умаров.

## Быстрый старт

### Требования

- **Quarto** (CLI).
- **Python 3** и зависимости из `requirements.txt` — для генерации рисунков и данных.
- **LibreOffice** актуальной версии — для рабочих книг Calc.
- **VS Code** с расширением Quarto — по желанию.
- Для сборки **PDF**: установленный LaTeX (TinyTeX/TeX Live). Рекомендуемый движок — **LuaLaTeX**. Для корректной кириллицы в PDF используются шрифты **Libertinus** (устанавливаются как TeX-пакет).

### Сборка и предпросмотр

Предпросмотр (live preview):

```bash
quarto preview
```

Сборка (HTML + PDF согласно `_quarto.yml`):

```bash
quarto render
```

Только HTML:

```bash
quarto render --to html
```

Только PDF:

```bash
quarto render --to pdf
```

### Быстрые проверки

Для небольших правок не требуется каждый раз собирать всю книгу. В проекте
есть единый сценарий проверки; его промежуточные результаты сохраняются в
исключённой из Git папке `tmp/check/`.

```bash
# Один изменённый раздел в HTML
scripts/check-book.sh quick book/chapters/01-random-events/05-interval-estimation.qmd

# Один раздел в выбранном формате
scripts/check-book.sh format book/chapters/01-random-events/05-interval-estimation.qmd pdf

# Текущая часть как сокращённая книга (сейчас — первая)
scripts/check-book.sh part html

# Полная контрольная сборка HTML + PDF
scripts/check-book.sh full
```

Режимы `quick` и `format` собирают раздел как самостоятельный документ: они
проверяют текст и локальную вёрстку, но не воспроизводят книжную навигацию,
сквозную нумерацию и межглавные ссылки. Режим `part` сохраняет устройство книги,
но ограничивает её текущей частью. Полная сборка по-прежнему обязательна после
смыслового этапа и перед выпуском версии.

В дальнейшем планируется также подготовить экспорт в редактируемые форматы
**DOCX** и **ODT**. Эти версии предназначены прежде всего для печати,
рецензирования и внесения правок в Microsoft Word или LibreOffice Writer;
пока официальными форматами публикации остаются HTML и PDF.

Очистка артефактов сборки:

```bash
rm -rf _book .quarto
```

## Установка TeX-зависимостей (шрифты Libertinus)

Если при сборке PDF появляются ошибки вида `font … not found`, установите пакет шрифтов:

```bash
tlmgr install libertinus-fonts
```

(Команда `tlmgr` доступна в окружении TinyTeX/TeX Live. На macOS может потребоваться добавить TeX в `PATH`.)

## Структура репозитория

```txt
.
├── _quarto.yml
├── _quarto-check.yml
├── _quarto-full.yml
├── _quarto-part.yml
├── book
│   ├── chapters
│   └── images
├── code
│   ├── data
│   ├── lib
│   ├── scripts
│   └── snippets
├── index.qmd
├── LICENSE.md
├── README.md
├── references.bib
├── scripts
│   └── check-book.sh
└── references.qmd

```

- `_quarto.yml` — общая конфигурация проекта Quarto Book (форматы вывода, параметры PDF/HTML, шрифты и т.п.).
- `_quarto-full.yml` — полный состав книги; этот профиль используется по умолчанию и в GitHub Actions.
- `_quarto-part.yml` — сокращённый состав текущей части с книжной навигацией и сквозными ссылками.
- `_quarto-check.yml` — самостоятельная быстрая сборка отдельных QMD-файлов в `tmp/check/`.
- `index.qmd` — входная страница книги (первый элемент `chapters`).
- `references.qmd` — страница списка литературы.
- `references.bib` — библиография в формате BibTeX.

### Книга (`book/`)

- `book/chapters/` — основной текст книги, главы сгруппированы по папкам.
  - В каждой папке главы — один файл `01-*.qmd` (раздел главы).
- `book/images/` — изображения, используемые в тексте книги (рисунки, схемы, экспортируемые диаграммы).

### Код и материалы (`code/`)

Единая папка для всего, что относится к воспроизводимым примерам и генерации артефактов.

- `code/snippets/` — **фрагменты, которые вставляются в книгу** (печатные примеры). Рекомендуемый формат — `.qmd`-вставки, подключаемые через include.
- `code/scripts/` — **служебные скрипты**: генерация рисунков/таблиц/датасетов. Результаты должны сохраняться в `book/images/` или `code/data/` (в зависимости от назначения).
- `code/lib/` — многократно используемые модули/утилиты (общие функции, которые вызываются из `scripts/` и/или `snippets/`).
- `code/data/` — небольшие входные/промежуточные данные для примеров и генераторов (только то, что разумно хранить в репозитории).

### Генерация материалов

Рисунки генерируются отдельными скриптами по главам и сохраняются в `book/images/`.
Зависимости устанавливаются один раз из корня проекта:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Примеры генерации:

```bash
python code/scripts/figures/ch01/generate_figures.py
python code/scripts/workbooks/ch01/create_interval_estimation_workbook.py
```

Временные результаты и диагностические файлы следует сохранять только в `tmp/`; эта папка исключена из Git.

### PDF-типографика и переносы

Настройки PDF вынесены в отдельные LaTeX-файлы:

- `book/tex/typography.tex` — типографика, переносы в формулах, абзацные отступы.
- `book/tex/hyphenation.tex` — пользовательские переносы слов.

### Инструкции и лицензия

- `LICENSE.md` — лицензия на материалы.
- `README.md` — описание проекта и инструкции для участников (данный файл).
- `CHANGELOG.md` — изменения опубликованных и готовящихся версий.
- `RELEASING.md` — правила нумерации и выпуска стабильных версий.

## Правила работы с изменениями

- Основная ветка `main` должна всегда собираться.
- Один коммит должен описывать одно законченное смысловое изменение.
- Текст, формулы, код и иллюстрации изменяются минимально; массовое переформатирование без необходимости не выполняется.
- Пути в проекте задаются относительно корня, а результаты генерации сохраняются только в `book/images/` или `code/data/`.
- Скрипты и примеры должны быть детерминированными; для случайных вычислений фиксируется начальное значение генератора.
- Артефакты сборки `_book/` и `.quarto/` не коммитятся.
## Публикация

Обычный push в `main` обновляет сайт книги и сохраняет PDF как артефакт сборки. Стабильный выпуск создаётся отдельным тегом вида `vX.Y.Z`; соответствующий релиз GitHub сохраняет собственный PDF и описание изменений. Благодаря этому актуальная сборка доступна на сайте, а выпуски остаются упорядоченными и воспроизводимыми.
